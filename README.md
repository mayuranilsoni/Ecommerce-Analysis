# E-Commerce Analytics Dashboard Project

## 📌 Project Overview

This project delivers an end-to-end data analytics solution designed to track, analyze, and optimize key performance indicators (KPIs) for an e-commerce platform. Using a relational database, complex SQL queries were constructed to extract deep business insights regarding customer behavior, payment preferences, shipping logistics, and review distributions.

The extracted datasets were structured and ultimately visualized across both **Power BI** and **Tableau Public** to build interactive, stakeholder-ready business intelligence dashboards.

---

## 🛠️ Tech Stack & Tools Used

* **Database / Querying:** SQL (Structured Query Language) — Complex Joins, CTEs (`WITH` clauses), Aggregations, Window Functions, and Date Manipulations.
* **Data Preparation & Staging:** Microsoft Excel (Master Table compilation, custom conditional data categorization).
* **Business Intelligence / Visualization:** * Microsoft Power BI Desktop
* Tableau Public



---

## 📐 Data Architecture & Schema Staging

The raw transactional data was consolidated into a single **Master Table** inside Excel to simplify the downstream ingestion pipeline for BI tools. Key transformations included:

* Extracting weekday/weekend flags from core timestamps.
* Mapping continuous values into categorical buckets (e.g., *Fast Delivery vs. Slow Delivery*, *Good Review vs. Bad Review*).

---

## 💻 SQL Analysis & Key Insights

The project answers critical operational and business questions through optimized SQL queries:

### 1. Payment Volume by Day Type (Weekday vs. Weekend)

* **Objective:** Understand if purchasing volume changes during weekends.
* **Insight:** Weekdays dominate revenue generation, accounting for roughly **78.18%** of total payment values ($15.38M) compared to weekends (**21.82%** / $4.29M).

### 2. High-Value Customer Experience Tracking

* **Objective:** Filter the count of successful orders that achieved a perfect review score (`5`) specifically using `credit_card` payments.

### 3. Logistical Efficiency for Specific Niches (`pet_shop`)

* **Objective:** Measure the average delivery times from purchase timestamp to customer delivery date exclusively for pet shop items to optimize regional supply chains.

### 4. Regional Revenue vs. Pricing Deep-Dive (São Paulo)

* **Objective:** Correlate individual product pricing variations against total actual payment values specifically generated by customers residing in `sao paulo` city.

### 5. Review Scores vs. Shipping Times Relationship

* **Objective:** Quantify if longer transit times directly result in poorer customer reviews.
* **Insight:** There is a clear inverse correlation. Perfect scores (`5`) maintain a lean average shipping delay of **10.6 days**, whereas poor scores (`1`) scale up significantly to **19.5 days**.

## 📊 Dashboard Implementations

### Option A: Power BI Dashboard (`Power_BI_Dashboard.pbix`)

Designed with a clean, dynamic neon-light theme prioritizing fluid usability:

* **KPI Scorecards:** Instant visibility into **Total Orders (115K)**, **Payment Value ($19.67M)**, **Avg Review Score (4.08)**, and **Total Customers (96K)**.
* **Advanced Slicers:** Left-hand navigation panel providing interactive dynamic filtering by `product_category_name`, `customer_city`, and `payment_type`.
* **Logistics Trend Line:** Visually maps out the drop-off in shipping days as review scores increase.

### Option B: Tableau Public Dashboard

A minimalist corporate layout emphasizing high macro-level view cross-filtering:

* **Payment Statistic Pie Chart:** Breaks down share-of-wallet distributions.
* **Dual-Axis Monthly Visuals:** Seamlessly overlays historical price vs. payment volumes side-by-side across seasonal cycles.

## 🚀 How to Replicate This Project

### Prerequisites

1. Install **MySQL** / **PostgreSQL** or any SQL IDE.
2. Install **Power BI Desktop**.
3. Create a free profile on **Tableau Public**.

### Execution Steps

1. **Database Setup:** Load the operational e-commerce relational schema into your local environment and execute the scripts provided in the `/SQL_Queries` directory.
2. **Reviewing Staging Layer:** Inspect the compiled `Master Table` within the `/Data` file to review the generated custom conditional categorization formulas.
3. **Run the Dashboard:** Open the `.pbix` file in Power BI Desktop or follow the URL link provided inside the Tableau text file to interactively explore the live, clickable dashboard instances.

## 📈 Key Recommendations Derived from Data

1. **Logistics Optimization:** Drastically reduce transit times exceeding 14 days, as any shipment taking longer than 13 days shows a strict statistical drop-off to review ratings below 3.0.
2. **Targeted Campaigns:** Focus marketing ad spends on mid-week cycles (Mon-Fri) where payment transaction volume spikes natively to maximize ROI.
